# 指标(Metrics)

**最锋利的一条:好指标度量的是用户得到的"结果",不是团队产出的"动作"——"上线了功能 X""发了 10 篇推送"是产出;"用户更快完成了任务""次周留存提升"才是结果。** 度量动作会让团队忙着交付却不创造价值。

## 结果 vs 产出(门 3 的核心)

- **产出(output)**:我们做了什么。可控,但不等于有用。
- **结果(outcome)**:用户行为/价值发生了什么可度量的变化。这才是目标。
- 每个 spec 的成功标准必须是结果。问:"如果功能上线了但这个数字没动,我们算成功吗?"——若答"不算",那个数字才是真指标。

## 好指标的判据

- **可度量**:能落到一个具体数字。"提升体验/活跃度"不是指标。
- **可归因**:这个数字动了,能合理归因到我们做的事吗?
- **可行动**:数字变差时,团队知道该去改什么。
- **抗操纵**:别选一个能靠损害用户来刷高的数(见护栏指标)。

## 北极星(North Star)

- 一个能代表"用户从产品中获得的核心价值"的指标,全队对齐用它。
- 它应**领先于**营收(是营收的因),且反映用户成功,例:协作工具的"周活跃协作文档数"。
- 北极星是方向,不是 KPI 暴政——别为了它牺牲长期健康。

## 护栏 / 反向指标(必配)

每个要推高的指标,都配一个**不能被牺牲**的护栏:

| 你在推 | 配套护栏(不能恶化) |
|------|------|
| 推送点击率 | 推送关闭率 / 卸载率 |
| 注册转化 | 次周留存(防骗注册) |
| 成交单量 | 退款率 / 投诉率 |
| 使用时长 | 用户主观满意度(防沉迷式增长) |

没有护栏的单指标优化,几乎一定会在别处制造伤害。门 3 不配护栏不算通过。

## 先行 vs 滞后

- **滞后指标**:留存、营收——重要,但变化慢、改不动当下。
- **先行指标**:激活完成率、首次价值时刻(time-to-value)——能预测滞后指标,且当下可影响。
- 团队日常盯**先行**指标(能行动),用**滞后**指标验证方向。

## OKR(目标与关键结果)

- **O(目标)**:定性、鼓舞、有方向。
- **KR(关键结果)**:量化的**结果**(不是任务清单),通常 3 个,有挑战性。
- 反模式:把 KR 写成"完成 A 功能"(那是任务/产出)。KR 应是"激活率从 X 到 Y"。

## HEART(体验类指标框架)

Happiness(满意)/ Engagement(参与)/ Adoption(新用户采纳)/ Retention(留存)/ Task success(任务成功率)。对每个维度先定 **Goal → Signal → Metric**,避免选了一堆度量不到目标的数。

## 危险信号

- 成功标准是"上线/完成功能" → 产出冒充结果,门 3 没过。
- 指标是"提升活跃度/体验更好" → 不可度量。
- 只有一个要推高的指标,没有护栏 → 一定在别处制造伤害。
- KR 写成任务清单 → 退回到结果表述。
- 全队只盯滞后指标(留存/营收) → 当下无从下手,补先行指标。
- 出现一个具体目标数字但没说依据 → 见铁律一,别凭空设 KPI。
