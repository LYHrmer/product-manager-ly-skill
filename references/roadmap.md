# 路线图(Roadmap)

**最锋利的一条:路线图是一份"结果"的承诺,不是"功能 + 日期"的清单——按时间排功能,会让团队在交付了一堆东西后,发现没人记得为什么、也没人对结果负责。** 路线图回答"何时、按什么次序、为达成什么结果",而 `prioritization.md` 回答"哪个赢"。

## 结果导向,而非功能清单

坏:`Q3:做 A 功能、B 功能;Q4:做 C 功能。`
好:`Now:把新用户首周留存从 X 提到 Y(候选手段:A/B);Next:降低报修流失;Later:探索传播。`

- 每个时间段挂一个**结果/指标**(连回 `metrics.md`),功能只是达成它的候选手段,可被替换。
- 这样当某个功能被证明无效时,团队会换手段而不是"反正做完了"。

## Now / Next / Later(优于精确季度甘特图)

- **Now**:正在做,已较确定,有明确结果。
- **Next**:下一步可能做,方向定了但细节待探索。
- **Later**:方向性意图,刻意保持模糊,**不承诺**。

越往后越模糊是**特性不是缺陷**——它诚实反映了不确定性,避免给出守不住的精确承诺。把 Later 画成带具体日期的甘特图,是在制造一定会被打脸的承诺。

## 排序的依据

时间次序不只看优先级分数,还看:

- **依赖**:A 是 B 的前置吗?(技术上、认知上、市场教育上)
- **学习价值**:先做哪个能最快撞实/撞碎最关键的假设?(连回 `evidence.md` 的假设登记)
- **风险前置**:把最不确定、最可能翻车的部分放前面,别留到最后才爆。
- **节奏与士气**:适当穿插能快速见效的项,维持团队和用户的信心。

## 对外沟通(路线图是沟通工具)

- 对不同受众讲不同粒度:高管要结果和战略对齐,工程要依赖和约束,用户/销售要价值和大致节奏(且明确"非承诺")。
- 永远标注**信心等级**:Now 高、Later 低。别让读者把方向当成签字画押。
- 变更要主动沟通:路线图一定会变,藏着不说比变更本身更伤信任。

## 危险信号

- 路线图是"功能 + 季度"网格,没有任何结果/指标 → 退回结果导向。
- Later 被画成带精确日期的承诺 → 制造必然失信。
- 排序只按优先级分数,忽略依赖和学习价值 → 可能先做了被后置项卡住的事。
- 最不确定的部分排在最后 → 风险没前置,后期容易爆雷。
- 给所有受众发同一张图同一粒度 → 沟通错位。
