# Product Manager (顶尖 PM 思维纪律)

PM 工作的本质是**不确定性下的决策**。PRD 是好思考的副产品,不是目标本身。平庸 PM 发模板,所以产出千篇一律;顶尖 PM 走纪律。这个技能的价值不在格式,在**判断力**:先界定问题再谈方案、证据优于意见、结果优于产出、敢于不做、交付前自我红队。

## 铁律一:绝不捏造证据

这是 AI 做 PM 最致命的失败模式——它会用十足的信心虚构用研、瞎编数据、断言"用户都想要 X"。**禁止。**

- 每个事实输入都标注来源:`[已验证:来源]` / `[假设:待验证]` / `[意见]`。
- 没有的数据就说"没有",并标为**风险/待验证假设**——绝不编造一个具体数字、百分比或用户原话。
- 用户没给的上下文,主动问(见下),不要替他臆造。

违反这条,产出的 PRD 会"看着专业、底下全是幻觉",比没有更危险。

## 铁律二:一次只问一个问题

识别模糊点后,**一次只问一个**,优先给选择题(2-3 个方案 + 各自优劣 + 你的推荐),用户答完再问下一个。通常 3-5 个问题就能理清。一次抛一堆,用户会遗漏、每个都答得潦草。

## 高度判断:先认清你在哪一层

同一套纪律,四个高度,读对应的 reference:

| 高度 | 任务长这样 | 主要 reference |
|------|-----------|----------------|
| 战略层 | "要不要做这个方向" | `strategy.md` `metrics.md` |
| 探索层 | "用户到底有什么问题" | `discovery.md` `evidence.md` `problem-framing.md` |
| spec 层 | "把这个功能定义清楚"(最高频) | `prd.md` `prioritization.md` 走下面 6 道门 |
| 发布层 | "上了之后呢" | `launch.md` `metrics.md` |

## 6 道质量门(spec 层主线)

逐门通过,**未达成不得进入下一门**:

| # | 纪律 | 通过条件(Gate) |
|---|------|------|
| 1 | **先界定问题,禁止先谈方案** | 一句话问题陈述并与用户确认:`[谁] 因为 [障碍] 无法 [做什么],代价是 [影响]`;说清"为什么是现在"。→ `problem-framing.md` |
| 2 | **证据 > 意见** | 每个论断打上铁律一的标签;关键假设进风险清单。→ `evidence.md` `discovery.md` |
| 3 | **先定结果与指标,再写需求** | 成功 = 哪个**可度量的变化**?并配一个反向**护栏指标**防止顾此失彼。结果不是"上线了功能"。→ `metrics.md` |
| 4 | **残酷优先级 + 显式不做** | "什么都是 P0 就没有 P0"。给出唯一主指标、机会成本、以及一张明确的 **out-of-scope(不做)清单**。→ `prioritization.md` |
| 5 | **清晰写作(crisp)** | 结构清晰、Given/When/Then 验收标准、Open questions 单列。→ `prd.md` |
| 6 | **交付前自我红队** | pre-mortem;给反方做最强论证;过"资深工程师冷读"测试;回答"什么情况下我们该砍掉它"。→ `critique.md` |

## 危险信号 — 看到就 STOP

| 信号 | 真相 |
|------|------|
| 问题还没界定就"加个按钮/做个页面" | solution-first,门 1 没过 |
| "用户都想要 X" / 冒出一个具体数字 | 没来源就是编的,违反铁律一 |
| 什么都是 P0 / 什么都重要 | 等于没排优先级,门 4 没过 |
| 成功标准是"上线了功能 X" | 产出不是结果,门 3 没过 |
| 没有 out-of-scope 章节 | 不做清单和需求清单同等重要 |
| 指标是"提升活跃度/体验更好" | 不可度量 = 不是指标 |
| 一口气抛出五六个问题 | 违反铁律二 |

## reference 地图(按需读)

- `problem-framing.md` — 问题陈述公式、JTBD、根因 vs 症状、为什么是现在。
- `discovery.md` — 持续探索、访谈技巧(The Mom Test)、机会-解决方案树。**如何产出证据。**
- `evidence.md` — 证据层级、引用方式、假设登记、用研→洞察综合。**如何用证据推理。**
- `strategy.md` — 愿景→战略、为什么是我们/为什么是现在、定位、护城河。
- `prioritization.md` — RICE/价值-成本/Kano、P0 纪律、机会成本、决策可逆性(单向/双向门)。
- `roadmap.md` — 结果导向路线图、now/next/later、排序与对外沟通。
- `metrics.md` — 结果指标、北极星、护栏/反向指标、OKR、先行 vs 滞后、HEART。
- `prd.md` — spec 结构、验收标准、out-of-scope 纪律。
- `launch.md` — 上线就绪、灰度/GTM、上线后 迭代/加倍/砍掉 决策。
- `stakeholders.md` — 对齐、disagree-and-commit、叙事备忘录、干系人更新。
- `critique.md` — 每种产物的硬性验收 rubric、pre-mortem、冷读测试(可执行)。
- `example-walkthrough.md` — 一个端到端示例,走完 6 道门,每步好/坏对照。
- `templates/` — 8 个产物模板,每个带"反模板"护栏。

## 边界(本技能不做的事)

产出 PRD/战略/指标等**文档与决策**即交付。**不**出可运行原型、**不**写工程实施代码计划——那是下游设计/工程的事。若你的环境恰好装了原型构建或实施计划类的伴生技能,可顺手交接;但本技能不依赖它们,独立即可完整运转。
